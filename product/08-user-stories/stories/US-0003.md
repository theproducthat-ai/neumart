## User Story

```
As an admin,
I want to see a Delivery section on the Order Detail page showing the current delivery status and task details,
So that I can track the delivery state of every order at a glance.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Section visible):**
  Given an order exists with a delivery task,
  When an admin opens Admin Order Detail (SCR-ADM-0009),
  Then a "Delivery" section is visible on the page showing the current delivery status.

- [ ] **AC-2 (Status = Pending shown):**
  Given an order was just placed (delivery task status = "pending"),
  When admin opens the order detail,
  Then the Delivery section shows status badge "Pending" with no assignment details.

- [ ] **AC-3 (Assigned details shown):**
  Given the delivery task has been assigned (status = "assigned", assignedTo = "Ravi Kumar"),
  When admin opens the order detail,
  Then the Delivery section shows "Assigned", the delivery person name, and the `assignedAt` timestamp.

- [ ] **AC-4 (Admin-only access):**
  Given a non-admin user (customer),
  When they attempt to access `/admin/orders/[id]`,
  Then they are redirected or shown an unauthorized error (existing admin guard — no change).

- [ ] **AC-5 (Real-time updates):**
  Given the admin has the order detail open,
  When delivery status is updated (by any admin session),
  Then the Delivery section updates without a page reload (Convex reactive query).
