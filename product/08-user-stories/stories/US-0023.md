## User Story

```
As an admin,
I want to see extended customer details when I scan a customer's QR code,
so that I can quickly look up their email, phone, and order history
without navigating to a separate admin screen.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Admin-enhanced view loads):**
  Given a signed-in admin opens `/qr/customer/{validQrCodeId}`
  When the page loads
  Then the page shows — in addition to the public fields (name, customerCode) — the customer's email (if set), phone (if set), total order count, last order date, and a link labelled "View in Admin"

- [ ] **AC-2 (Email / phone absent gracefully):**
  Given a customer has no phone number set on their Convex record
  When an admin opens their QR scan result page
  Then the phone row is omitted silently — no empty field, no placeholder shown

- [ ] **AC-3 (Admin link navigates correctly):**
  Given an admin is on the QR scan result page
  When they click "View in Admin"
  Then they are navigated to the admin order list filtered to that customer, or the admin dashboard if no customer-specific filter exists

- [ ] **AC-4 (Admin role required — not just any logged-in user):**
  Given a signed-in customer (non-admin) opens a QR scan result page
  When the page loads
  Then only the public view is shown — the enhanced admin fields are not visible

- [ ] **AC-5 (Admin signs out — view degrades to public):**
  Given an admin was viewing the enhanced scan result, then signs out and refreshes
  When the page reloads
  Then only the public view is shown (name + customerCode) — enhanced fields disappear

- [ ] **AC-6 (Disabled QR — admin view):**
  Given a customer's `qrEnabled` is `false`
  When an admin opens that QR scan result URL
  Then the disabled message is shown — even the admin view reveals no customer data for a disabled QR

## Business Rules

| # | Rule |
|---|---|
| BR-5 | Admin-enhanced fields (email, phone) must be served via an authenticated query — not the public `getCustomerByQrCode` query (PRD BR-5) |
| BR-6 | Disabled QR returns no customer data regardless of the viewer's role (PRD BR-6) |

---

## Validation Rules

| Field | Rule | Error Message |
|---|---|---|
| Admin role check | Viewer must have Clerk `publicMetadata.role === "admin"` | Fall back to public view — no error message |
| `qrEnabled` | Must be `true` even for admin | "This QR code has been disabled." |
