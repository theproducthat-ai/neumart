## User Story

```
As a Customer,
I want the banner carousel to automatically advance to the next banner every 4 seconds,
So that I see all current promotions without having to manually interact.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Auto-scroll advances banners):**
  Given the carousel has 3 or more active banners,
  When 4 seconds pass without user interaction,
  Then the carousel advances to the next banner.

- [ ] **AC-2 (Auto-scroll loops):**
  Given the carousel is on the last banner,
  When 4 seconds pass,
  Then the carousel wraps back to the first banner.

- [ ] **AC-3 (Auto-scroll disabled for single banner):**
  Given exactly one active banner exists,
  When the page loads,
  Then no auto-scroll interval is started.

- [ ] **AC-4 (Pause on desktop hover):**
  Given the carousel has multiple active banners and is auto-scrolling,
  When the customer moves their cursor over the carousel,
  Then the auto-scroll pauses and the current banner stays visible.

- [ ] **AC-5 (Resume after desktop hover ends):**
  Given the customer has hovered over the carousel and paused auto-scroll,
  When the customer moves their cursor away from the carousel,
  Then auto-scroll resumes from the current banner.

- [ ] **AC-6 (Pause on mobile touch):**
  Given the carousel has multiple active banners and is auto-scrolling,
  When the customer touches the carousel on a mobile device,
  Then the auto-scroll pauses.

- [ ] **AC-7 (No stale interval on unmount):**
  Given the customer navigates away from `/products` while auto-scroll is running,
  When the component unmounts,
  Then the interval is cleared and no console errors or memory leaks occur.
