## User Story

```
As an admin,
I want to enter allergen and ingredient details when creating a new product,
So that customers can see accurate safety information on the storefront.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Fields present on form):**
  Given an admin navigates to `/admin/products/new`,
  When the form loads,
  Then an "Allergen & Ingredient" section is visible at the bottom with 5 fields: Ingredients (textarea), Contains Allergens (multi-select), May Contain Allergens (multi-select), Dietary Tags (multi-select), Allergen Notes (textarea)

- [ ] **AC-2 (Save with all allergen fields populated):**
  Given an admin fills in all 5 allergen fields with valid data,
  When the form is submitted,
  Then the product is created and all 5 allergen fields are persisted correctly

- [ ] **AC-3 (Save with no allergen fields):**
  Given an admin fills in only the required product fields (name, price, etc.) and leaves all allergen fields blank,
  When the form is submitted,
  Then the product is created without error and allergen fields are absent

- [ ] **AC-4 (Multi-select is fixed list only):**
  Given an admin opens the Contains Allergens multi-select,
  When they interact with it,
  Then only the 12 allergen values are available — no freetext input is possible

- [ ] **AC-5 (Dietary tags fixed list only):**
  Given an admin opens the Dietary Tags multi-select,
  When they interact with it,
  Then only the 10 dietary tag values are available — no freetext input is possible

- [ ] **AC-6 (Multiple selections save correctly):**
  Given an admin selects "Milk" and "Wheat" from Contains Allergens,
  When the form is submitted,
  Then `containsAllergens` is saved as `["Milk", "Wheat"]`

---

## Business Rules

| # | Rule |
|---|---|
| BR-1 | All 5 allergen fields are optional — product saves without any of them |
| BR-2 | `containsAllergens` multi-select values must come from the fixed 12-item allergen list only |
| BR-3 | `mayContainAllergens` multi-select values must come from the same fixed 12-item allergen list |
| BR-4 | `dietaryTags` multi-select values must come from the fixed 10-item dietary tag list only |
| BR-5 | `ingredients` and `allergenNotes` are freeform textarea — no character limit enforced in MVP |

---

## Validation Rules

| Field | Rule | Error Message |
|---|---|---|
| Contains Allergens | Values must be from fixed allergen list; enforced by multi-select UI | Not applicable — freetext not possible |
| May Contain Allergens | Values must be from fixed allergen list | Not applicable |
| Dietary Tags | Values must be from fixed dietary tag list | Not applicable |
| All allergen fields | Optional — no required validation | — |
