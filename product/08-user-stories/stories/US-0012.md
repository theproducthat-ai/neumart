## User Story

```
As a Customer on a desktop browser,
I want to see arrow buttons on either side of the carousel and dot indicators below it,
So that I can manually navigate between banners and know how many promotions there are.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Arrows visible on desktop):**
  Given the carousel has 2 or more active banners,
  When the page is viewed on a desktop viewport (≥ `md` breakpoint),
  Then left and right arrow buttons are visible on the carousel.

- [ ] **AC-2 (Right arrow advances):**
  Given the customer is viewing any banner that is not the last,
  When the customer clicks the right arrow,
  Then the carousel advances to the next banner.

- [ ] **AC-3 (Right arrow on last banner wraps):**
  Given the customer is on the last banner,
  When the customer clicks the right arrow,
  Then the carousel wraps to the first banner.

- [ ] **AC-4 (Left arrow goes back):**
  Given the customer is viewing any banner that is not the first,
  When the customer clicks the left arrow,
  Then the carousel goes back to the previous banner.

- [ ] **AC-5 (Left arrow on first banner wraps):**
  Given the customer is on the first banner,
  When the customer clicks the left arrow,
  Then the carousel wraps to the last banner.

- [ ] **AC-6 (Arrows hidden on mobile):**
  Given the page is viewed on a mobile viewport (below `md` breakpoint),
  When the carousel is rendered,
  Then the arrow buttons are not visible.

- [ ] **AC-7 (Dot indicators reflect active banner):**
  Given the carousel has N active banners,
  When any banner is displayed,
  Then N dots are visible, and the dot corresponding to the active banner is visually distinct (e.g. filled/darker).

- [ ] **AC-8 (Clicking a dot navigates to that banner):**
  Given the carousel is showing banner 1,
  When the customer clicks the dot for banner 3,
  Then the carousel shows banner 3 and dot 3 becomes the active dot.

- [ ] **AC-9 (No arrows or dots for single banner):**
  Given exactly one active banner exists,
  When the page loads,
  Then no arrow buttons and no dot indicators are rendered.

## Business Rules

| # | Rule |
|---|---|
| BR-1 | Arrows are shown only on `md` and above breakpoints — hidden on mobile via Tailwind (`hidden md:flex`) |
| BR-2 | Dots are shown on all screen sizes when there are 2 or more banners |
| BR-3 | Active dot must be visually distinct from inactive dots |
| BR-4 | Arrow clicks reset the auto-scroll timer to prevent immediate auto-advance after manual navigation |
