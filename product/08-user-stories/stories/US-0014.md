## User Story

```
As a Customer,
I want to click or tap a promotional banner and be taken to the relevant page,
So that I can immediately browse the offer or category being promoted without extra navigation steps.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Click navigates to linkTarget):**
  Given a banner has a valid `linkTarget` (e.g. `/products?category=fruits`),
  When the customer clicks or taps the banner,
  Then the customer is navigated to that route using client-side routing (no full page reload).

- [ ] **AC-2 (No navigation for empty linkTarget):**
  Given a banner has an empty string or undefined `linkTarget`,
  When the customer clicks or taps the banner,
  Then nothing happens — no navigation, no error thrown, no console warning.

- [ ] **AC-3 (Cursor indicates clickability):**
  Given a banner has a valid non-empty `linkTarget`,
  When the customer hovers over it on desktop,
  Then the cursor changes to a pointer (`cursor-pointer`).

- [ ] **AC-4 (No cursor change for non-clickable banner):**
  Given a banner has an empty or undefined `linkTarget`,
  When the customer hovers over it on desktop,
  Then the cursor does not change to a pointer.

- [ ] **AC-5 (Client-side routing — no full reload):**
  Given a customer clicks a banner with `linkTarget: "/products"`,
  When the navigation completes,
  Then the page does not perform a full browser reload — the Next.js router handles navigation.

---

## Business Rules

| # | Rule |
|---|---|
| BR-1 | Only internal routes are supported — `linkTarget` must start with `/` |
| BR-2 | External URLs (`http://`, `https://`) are not supported in MVP — no `<a target="_blank">` |
| BR-3 | Navigation uses Next.js `router.push()` or `<Link>` — not `window.location.href` |
| BR-4 | A banner with an empty or undefined `linkTarget` is rendered but non-interactive |
