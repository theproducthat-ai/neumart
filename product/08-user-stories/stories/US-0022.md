## User Story

```
As a store staff member or any person with a customer's QR code,
I want to scan the QR code and see the customer's name and Customer ID,
so that I can confirm their identity quickly without asking for personal details.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Valid QR — public view):**
  Given a visitor (unauthenticated) opens `/qr/customer/{validQrCodeId}`
  When the page loads
  Then the label "Nuemart Customer", the customer's name, and "Customer ID: CUST-000124" are displayed — nothing else

- [ ] **AC-2 (No sensitive data exposed):**
  Given a valid QR scan result page is loaded by an unauthenticated visitor
  When the page is fully rendered
  Then no email address, phone number, delivery address, order history, Convex `_id`, or Clerk `tokenIdentifier` is present anywhere on the page or in the page's network response

- [ ] **AC-3 (Invalid qrCodeId):**
  Given a visitor opens `/qr/customer/nonexistent-code`
  When the page loads
  Then the message "QR code not found." is displayed — no customer data is shown

- [ ] **AC-4 (Disabled QR):**
  Given a customer's `qrEnabled` is `false`
  When a visitor opens that customer's QR scan result URL
  Then the message "This QR code has been disabled." is displayed — no customer name or ID is shown

- [ ] **AC-5 (No auth required):**
  Given a visitor is not signed in to Nuemart
  When they open a valid QR scan result URL
  Then the public view loads without any sign-in prompt or redirect

- [ ] **AC-6 (getCustomerByQrCode query field restriction):**
  Given the `getCustomerByQrCode` Convex query is called with a valid `qrCodeId`
  Then the query returns only `name`, `customerCode`, and `qrEnabled` — never `email`, `phone`, `role`, `tokenIdentifier`, or `_id`

## Business Rules

| # | Rule |
|---|---|
| BR-5 | `getCustomerByQrCode` must never return `email`, `phone`, `role`, `tokenIdentifier`, or Convex internal ID to an unauthenticated caller (PRD BR-5) |
