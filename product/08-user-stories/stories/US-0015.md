## User Story

```
As a system,
I want the products table to support allergen and ingredient fields,
So that admin can store this data and customers can view it.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Schema deployed):**
  Given the updated schema is deployed via `npx convex deploy`,
  When a product is fetched,
  Then the response shape includes the 5 new optional fields: `ingredients`, `containsAllergens`, `mayContainAllergens`, `dietaryTags`, `allergenNotes`

- [ ] **AC-2 (Existing products unaffected):**
  Given existing products were created before this change,
  When those products are fetched,
  Then they return successfully with all existing fields intact and the 5 new fields as `undefined`

- [ ] **AC-3 (Optional fields accept undefined):**
  Given a product is created with no allergen fields provided,
  When the product is saved,
  Then it saves without error and the 5 fields are absent from the record

- [ ] **AC-4 (Array fields accept multiple values):**
  Given a product is created with `containsAllergens: ["Milk", "Wheat"]`,
  When the product is fetched,
  Then `containsAllergens` returns `["Milk", "Wheat"]` exactly

---

## Business Rules

| # | Rule |
|---|---|
| BR-1 | All 5 fields must be optional (`v.optional`) — no product create or update should fail due to absent allergen fields |
| BR-2 | `containsAllergens` and `mayContainAllergens` must be typed as `v.optional(v.array(v.string()))` — not a single string |
| BR-3 | `dietaryTags` must be typed as `v.optional(v.array(v.string()))` |
| BR-4 | `ingredients` and `allergenNotes` must be typed as `v.optional(v.string())` |
