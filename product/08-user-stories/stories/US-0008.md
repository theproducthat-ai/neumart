## User Story

```
As a customer,
I want to see the current delivery status of my order on my Order Detail page,
So that I know whether my order is being prepared, out for delivery, or has been delivered — without calling the store.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Section visible to customer):**
  Given a customer has placed an order,
  When they open their Order Detail page (SCR-CUS-0010),
  Then a "Delivery" section is visible showing the current delivery status.

- [ ] **AC-2 (Initial status — Pending):**
  Given the order was just placed (delivery task status = "pending"),
  When customer views the Order Detail,
  Then Delivery section shows status label "Pending" and the delivery address.

- [ ] **AC-3 (Status updates in real time):**
  Given admin updates delivery status to "out_for_delivery",
  When customer views their Order Detail (or has it open),
  Then the Delivery section updates to show "Out for Delivery" without a page reload.

- [ ] **AC-4 (No delivery person name shown):**
  Given a delivery task has been assigned with `assignedTo = "Ravi Kumar"`,
  When customer views the Order Detail,
  Then the delivery person name is NOT visible on the page.

- [ ] **AC-5 (Failed status shown safely):**
  Given delivery status = "failed",
  When customer views their Order Detail,
  Then the Delivery section shows "Delivery Failed". No reason code or admin notes are shown.

- [ ] **AC-6 (Cancelled status shown safely):**
  Given delivery status = "cancelled",
  When customer views their Order Detail,
  Then the Delivery section shows "Delivery Cancelled". No reason shown.

- [ ] **AC-7 (Server-side authorization):**
  Given customer A and customer B each have their own orders,
  When customer A's session queries `getCustomerDeliveryStatus` with customer B's orderId,
  Then the query returns null — no data is exposed.

- [ ] **AC-8 (Existing page content unchanged):**
  Given a customer opens their Order Detail,
  Then existing content (order items, totals, payment status, order status) is all still visible and unchanged.

## Business Rules

| # | Rule |
|---|---|
| BR-1 | Customer sees: delivery status label, delivery address only |
| BR-2 | Customer does NOT see: delivery person name, contact, internal notes, failedReasonCode, failedReasonNotes, any admin-only field |
| BR-3 | The Convex query `getCustomerDeliveryStatus` must verify `deliveryTask.userId === ctx.auth user's ID` server-side |
| BR-4 | `userId` must never be passed from the client — resolved from `ctx.auth` (DEC-003) |
| BR-5 | The section is read-only for customers — no actions available |
| BR-6 | Section must be additive — no existing content on SCR-CUS-0010 is removed |

---

## Validation Rules

| Field | Rule | Error Message |
|---|---|---|
| Order ownership | `deliveryTask.userId` must match authenticated user's ID | Query returns null silently — no error message exposed to customer |
| Auth required | Customer must be authenticated | Existing auth guard on `/orders/[id]` applies |
