## User Story

```
As a customer,
I want to view my personal QR code on my profile page,
so that I can show it to store staff or download it for later use.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Page renders correctly):**
  Given a signed-in customer navigates to `/account/profile`
  When the page loads
  Then the page heading "My Customer QR Code" is visible, along with the customer's name, email, Customer ID (format: `CUST-000124`), and a rendered QR code image

- [ ] **AC-2 (QR encodes correct URL):**
  Given the QR code is rendered on the profile page
  When a phone scans the QR code
  Then the browser opens `https://neumart.com/qr/customer/{qrCodeId}` for that customer

- [ ] **AC-3 (Generating state ‚Äî first visit):**
  Given a customer visits `/account/profile` and their user record has no `qrCodeId`
  When the page loads
  Then a loading / "Generating your QR code..." state is shown while `generateQrCode` mutation runs, and the QR code appears once the mutation completes

- [ ] **AC-4 (Download QR):**
  Given the profile page is loaded with a QR code displayed
  When the customer clicks "Download QR"
  Then a QR code image file is saved to their device

- [ ] **AC-5 (Auth guard):**
  Given an unauthenticated visitor navigates to `/account/profile`
  When the page loads
  Then the visitor is redirected to the sign-in page ‚Äî the profile page is never shown

- [ ] **AC-6 (Account menu link):**
  Given a signed-in customer views the account navigation menu
  When the menu is open
  Then a link labelled "My QR Code" is present and navigates to `/account/profile`

- [ ] **AC-7 (No qrCodeId edge case ‚Äî mutation failure):**
  Given `generateQrCode` mutation fails or times out
  When the page is in the generating state
  Then an error message or retry option is shown ‚Äî the page does not crash
