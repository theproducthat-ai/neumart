## User Story

```
As a Customer,
I want to see a promotional banner carousel at the top of the product listing page,
So that I can discover current offers and featured categories as soon as I land on the storefront.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Carousel renders at top):**
  Given the customer is on `/products` and at least one banner has `isActive: true`,
  When the page loads,
  Then the carousel renders above the category filter bar and the product grid.

- [ ] **AC-2 (Component accepts prop):**
  Given the `<BannerCarousel>` component,
  When it is rendered,
  Then it accepts `banners: BannerConfig[]` as a prop and does not import the config file internally.

- [ ] **AC-3 (isActive filtering applied at parent):**
  Given the product listing page imports the static config,
  When it passes banners to `<BannerCarousel>`,
  Then only banners with `isActive: true` are passed — filtering happens before the component, not inside it.

- [ ] **AC-4 (Zero active banners — no render):**
  Given all banners have `isActive: false`,
  When the product listing page loads,
  Then the carousel section is not rendered and no empty space or placeholder appears above the product grid.

- [ ] **AC-5 (One active banner — single image shown):**
  Given exactly one banner has `isActive: true`,
  When the page loads,
  Then that banner is displayed with no arrows and no dot indicators.

- [ ] **AC-6 (Broken image fallback):**
  Given a banner whose `imageUrl` returns a 404 or fails to load,
  When the carousel renders,
  Then the banner slot shows a solid fallback background colour (e.g. `bg-muted`) and no broken image icon is visible.

- [ ] **AC-7 (Responsive layout — rounded corners and spacing):**
  Given the carousel is rendered on any screen size,
  When the customer views it,
  Then the banners have rounded corners, correct horizontal padding, and the aspect ratio does not shift on image load.

- [ ] **AC-8 (Mobile-first rendering):**
  Given the customer is on a mobile viewport,
  When the page loads,
  Then the carousel fills the full content width with appropriate spacing and rounded corners.

## Business Rules

| # | Rule |
|---|---|
| BR-1 | `<BannerCarousel>` must accept `banners` as a prop — never import config directly |
| BR-2 | isActive filtering happens in the parent page, not inside the carousel component |
| BR-3 | Carousel does not render at all when no active banners are passed |
| BR-4 | Banner container must use a fixed aspect ratio (16:5 recommended) to prevent layout shift |
