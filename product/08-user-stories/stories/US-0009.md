## User Story

```
As a System,
I want a typed BannerConfig definition and a static config array with placeholder banners,
So that the carousel has structured, type-safe content to render in MVP without requiring a database.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Type definition exists):**
  Given the project is built with TypeScript,
  When a developer imports `BannerConfig` from the config file,
  Then all required fields (`id`, `imageUrl`, `linkTarget`, `isActive`) are enforced by the type and optional fields (`title`, `subtitle`, `ctaText`) are typed correctly.

- [ ] **AC-2 (Config array is populated):**
  Given the static config file exists,
  When a developer reads the banners array,
  Then it contains between 3 and 5 banner entries with representative placeholder content.

- [ ] **AC-3 (isActive filtering works):**
  Given the config contains banners with mixed `isActive` values,
  When the config is filtered to only `isActive: true`,
  Then only the active banners remain.

- [ ] **AC-4 (No TypeScript errors):**
  Given any banner entry in the config,
  When the project is type-checked (`pnpm typecheck`),
  Then no TypeScript errors are reported for the config file.

- [ ] **AC-5 (Scalability contract):**
  Given the `BannerConfig` type,
  When compared to the fields documented in PRD-0002 Data Requirements,
  Then all fields match exactly — `id`, `imageUrl`, `title?`, `subtitle?`, `ctaText?`, `linkTarget`, `isActive`.
