## User Story

```
As a customer,
I want to see a full allergen and ingredient breakdown on the product detail page,
So that I can make an informed decision about whether this product is safe and suitable for me before purchasing.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Section always renders):**
  Given any product detail page,
  When a customer views it,
  Then an "Allergen & Ingredient Information" section heading is always present — it is never hidden entirely

- [ ] **AC-2 (Full data — all sub-sections shown):**
  Given a product has all 5 allergen fields populated,
  When a customer views the product detail page,
  Then all 5 sub-sections are rendered: Ingredients, Contains, May Contain Traces Of, Dietary Information, Notes

- [ ] **AC-3 (Partial data — only populated sub-sections shown):**
  Given a product has only `ingredients` and `dietaryTags` set,
  When a customer views the product detail page,
  Then only "Ingredients" and "Dietary Information" sub-sections render — no empty "Contains" or "May Contain" headers are shown

- [ ] **AC-4 (No data — fallback message):**
  Given a product has no allergen data (all 5 fields absent),
  When a customer views the product detail page,
  Then the section header renders with only the message: "Allergen information not available."

- [ ] **AC-5 (No affirmative safety claim):**
  Given any product regardless of allergen data state,
  When a customer views the product detail page,
  Then the text "No allergens", "Allergen-free", or any equivalent safety claim is never displayed

- [ ] **AC-6 (Dietary tags only — no allergen sub-sections):**
  Given a product has only `dietaryTags: ["Vegan", "Organic"]` and no allergen fields,
  When a customer views the product detail page,
  Then only the "Dietary Information" sub-section renders showing "Vegan, Organic"

- [ ] **AC-7 (Section position):**
  Given a product detail page,
  When the page loads,
  Then the Allergen & Ingredient Information section appears below the product description

## Business Rules

| # | Rule |
|---|---|
| BR-1 | The section heading "Allergen & Ingredient Information" always renders |
| BR-2 | Each sub-section renders only when its corresponding field has data |
| BR-3 | When all 5 fields are absent/empty → show only the heading and "Allergen information not available." |
| BR-4 | "No allergens", "Allergen-free", or any affirmative claim must never be auto-generated |
| BR-5 | `containsAllergens` and `mayContainAllergens` values are displayed as a comma-separated list |
| BR-6 | `dietaryTags` values are displayed as a comma-separated list or as individual tag chips |
| BR-7 | `ingredients` and `allergenNotes` values are displayed as plain text |
| BR-8 | Empty arrays (`[]`) are treated the same as absent — the sub-section does not render |
