## User Story

```
As a system,
I want a deliveryTasks table in the Convex schema with all required fields and an index,
So that delivery task data can be stored, queried, and updated for every order.
```

---

## Acceptance Criteria

- [ ] **AC-1 (Schema exists):**
  Given the schema is deployed,
  When Convex schema is inspected,
  Then a `deliveryTasks` table exists with all 19 fields defined per PRD-0001 Data Requirements.

- [ ] **AC-2 (Status is a literal union):**
  Given the schema is deployed,
  When a delivery task is created with an invalid status string,
  Then Convex rejects the insert at the schema level (TypeScript compile error or runtime validator error).

- [ ] **AC-3 (failedReasonCode is a literal union):**
  Given the schema is deployed,
  When a delivery task is created with an invalid `failedReasonCode` value,
  Then Convex rejects the insert at the schema level.

- [ ] **AC-4 (Index exists):**
  Given the schema is deployed,
  When `ctx.db.query("deliveryTasks").withIndex("by_orderId", q => q.eq("orderId", id))` is called,
  Then it resolves without error and returns results efficiently.

## Business Rules

| # | Rule |
|---|---|
| BR-1 | `status` field must be defined as a Convex literal union: `"pending" \| "assigned" \| "picked_up" \| "out_for_delivery" \| "delivered" \| "failed" \| "cancelled"` |
| BR-2 | `failedReasonCode` must be defined as a Convex literal union: `"customer_unavailable" \| "wrong_address" \| "customer_refused" \| "delivery_person_unavailable" \| "order_damaged" \| "payment_issue" \| "unable_to_contact" \| "other"` |
| BR-3 | `by_orderId` index must be defined on the `orderId` field to support efficient order-based lookups |
| BR-4 | `updatedBy` must be typed as `Id<"users">`, not a raw string |

---

## Validation Rules

| Field | Rule | Error Message |
|---|---|---|
| `status` | Must match literal union | TypeScript / Convex schema compile error |
| `failedReasonCode` | Must match literal union or be undefined | TypeScript / Convex schema compile error |
| `orderId` | Must be a valid `Id<"orders">` | TypeScript type error |
| `userId` | Must be a valid `Id<"users">` | TypeScript type error |
| `addressId` | Must be a valid `Id<"addresses">` | TypeScript type error |
| `updatedBy` | Must be a valid `Id<"users">` | TypeScript type error |
