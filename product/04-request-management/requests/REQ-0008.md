## Request Summary

Add visual icons to each category filter pill on the product listing page. Current pills (All, Fruits, Vegetables, Dairy, Bakery, Beverages, Snacks, Staples, Household) show text only. The request is to add an icon — emoji or icon library glyph — specific to each category.

---

## Source Request

> "Add icons specific to each category."

User provided a screenshot of the category filter pills row on the product listing page.

---

## Grilling Notes

Grilling in progress. Open questions:

1. **Icon placement** — Should the icon appear to the left of the category label text, or should it replace the text entirely? (e.g. "🍎 Fruits" vs just "🍎")
2. **Icon style** — What style of icons? Options: emoji (🍎🥕🥛), a React icon library (Lucide, React Icons), or custom SVG assets?
3. **"All" pill** — Should the "All" pill also get an icon (e.g. a grid or all-items icon), or stay text-only?
4. **Admin-configurable or hardcoded?** — Should the admin be able to change category icons from the admin console, or is a hardcoded mapping per category name acceptable?

---

## Scope Notes

- No schema change required if icons are hardcoded per category name in the frontend.
- Schema change (add `icon` field to `categories` table) would be needed if icons must be admin-configurable.
- The category filter component lives in the Customer Commerce / Product Listing screen (SCR-CUS-0001).
