## Request Summary

Add allergen-specific and ingredient-level details to every product. Customers should be able to view what ingredients are present in a product and whether it contains common allergens. The information should be visible on the customer-facing product experience and manageable from the admin side when creating or editing products.

## Business Context

Products currently show only basic information: name, quantity, price, image, stock status, and add-to-cart. Customers may want to avoid certain ingredients due to allergy, health, diet, religious, lifestyle, or personal preference reasons. This is a product transparency enhancement.

## Fields Required Per Product

### A. Ingredients
Freeform text listing actual ingredients used in the product.
Example: "Wheat flour, sugar, edible vegetable oil, milk solids, cocoa powder, raising agent, salt"

### B. Contains Allergens
Multi-select from a fixed allergen list. Known allergens directly present in the product.
Example: Milk, Wheat, Soy

### C. May Contain (Trace Allergen Warning)
Multi-select from a fixed allergen list. Allergens that may be present due to shared facility or equipment cross-contamination.
Example: Peanuts, Tree nuts

### D. Allergen-Free Claims
Optional tags shown only when explicitly set by admin (Gluten-free, Dairy-free, Nut-free, Soy-free). App must not auto-infer these.

### E. Dietary Tags
Optional multi-select: Vegetarian, Vegan, Jain-friendly, Organic, No added sugar, Low fat.

### F. Allergen Notes
Freeform text for additional manufacturer or processing notes.
Example: "Manufactured in a facility that also processes nuts."

## Suggested Fixed Allergen List
Milk, Peanuts, Tree nuts, Wheat, Gluten, Soy, Sesame, Mustard, Egg, Fish, Shellfish, Sulphites

## Customer-Facing Experience

### Product Listing — SCR-CUS-0001
- Show a small allergen badge on the product card only when allergens are present. No badge = no data or no allergens.
- Do not show the full ingredient list on the card (would clutter).
- Recommended badge text: "Contains: Peanut" or "Contains allergens"

### Product Detail — SCR-CUS-0002
- Full "Product Information" section placed below product description.
- Sections: Ingredients, Allergen Information (Contains / May Contain), Dietary Information.
- If no allergen data is present, section is hidden or shows a neutral state.

## Admin-Side Experience

### New Product — SCR-ADM-0006 / Edit Product — SCR-ADM-0007
New fields added to the product form:
- Ingredients (textarea — freeform)
- Contains Allergens (multi-select from fixed list)
- May Contain Allergens (multi-select from fixed list)
- Dietary Tags (multi-select from fixed list)
- Allergen Notes (textarea — freeform)

## Schema Impact

New optional fields required on the `products` table in `schema.ts`:
- `ingredients` — optional string
- `containsAllergens` — optional array of strings
- `mayContainAllergens` — optional array of strings
- `dietaryTags` — optional array of strings
- `allergenNotes` — optional string

All fields are optional. Existing products will have no allergen data and the UI will handle this gracefully.

## Grilling Answers (2026-06-23)

### Q1 — Optional vs required on admin form
All allergen fields are **optional**. Admin can save a product with all allergen fields blank. Existing products will have no allergen data and must not break.

Customer-facing fallback rule:
- If any allergen data exists → show the Allergen & Ingredient section with available data.
- If no allergen data exists → show "Allergen information not available." Do NOT hide the section entirely.
- NEVER auto-infer "No allergens" or "Allergen-free" — only show these if explicitly set by admin.

Future enhancement (not MVP): admin-side completeness indicator (Missing / Added / Needs review).

### Q2 — Dietary tags
Fixed multi-select list (not freeform). MVP list:
Vegetarian, Vegan, Jain-friendly, Organic, No added sugar, Low fat, Gluten-free, Dairy-free, Nut-free, Soy-free.

Admin can select multiple. Custom tags are a future enhancement via an admin-managed tag master.

### Q3 — Allergen badge on product card
Generic badge, not a named list. Rules:
- `containsAllergens` present → **"Contains allergens"**
- Only `mayContainAllergens` present → **"May contain allergens"**
- Both present → **"Contains allergens"** (containsAllergens takes precedence)
- No allergen data → **no badge** (do not show "No allergens")

### Q4 — Filtering out of scope for MVP
Allergen and dietary tag filters on the listing page are **out of scope** for this request. Schema must store allergens and tags as structured arrays (not text blobs) to support future filtering. No filter UI to be built now.

## MVP Scope (Confirmed)

1. Optional allergen and ingredient fields on admin product create/edit form.
2. Structured schema fields: `ingredients`, `containsAllergens`, `mayContainAllergens`, `dietaryTags`, `allergenNotes`.
3. Fixed dietary tag multi-select from defined list.
4. Generic allergen badge on product listing card (Contains allergens / May contain allergens).
5. Full allergen and ingredient section on product detail page.
6. Fallback message: "Allergen information not available." when no data exists.
7. No listing page allergen/dietary filters in MVP.
8. Schema future-ready for filtering (array fields, not strings).
