## Request Summary

Give each registered customer a unique, permanent QR code tied to a safe public identifier (not a raw Convex user ID). The QR code lives on a new customer profile page and, when scanned, opens a lightweight Customer ID / Loyalty profile page showing limited customer details. Store staff and admins scan it to identify the customer for loyalty, order lookup, or support. Customers can view and download their QR code.

## Business Context

Nuemart needs a way for store staff to quickly identify a customer without asking for their name or phone number. A scannable customer QR code bridges the in-store and digital experience. It also positions Nuemart for future loyalty, wallet, and delivery verification use cases without over-engineering the MVP.

## Grilling Answers (2026-06-23)

### Q1 — What does scanning the QR code do?

Opens a lightweight **Customer ID / Loyalty profile page** with limited information.

| Field | Shown to Public / Unauth | Shown to Admin/Staff |
|---|---|---|
| Customer name | Yes | Yes |
| Customer ID (customerCode) | Yes | Yes |
| Phone number | Masked only | Full (if available) |
| Membership status | Yes | Yes |
| Loyalty points | Optional (later) | Yes (if built) |
| Order count | No | Yes |
| Last order | No | Yes |
| Full address | No | No |
| Recent orders list | No | No |
| Wallet balance | No | Optional (later) |

The QR is a **Customer ID / Loyalty QR**, not a general public profile.

### Q2 — Who scans it?

**MVP:** Store staff and admin only.

| Scanner | MVP? | Use case |
|---|---|---|
| Store staff | Yes | Identify customer, apply loyalty, lookup order |
| Admin | Yes | Quick customer lookup from any device |
| Delivery person | Later | Verify customer/order at delivery |
| Customer (to share) | Later | Share membership |

### Q3 — Downloadable or shareable?

**MVP:** Display + download as image.

| Feature | Scope |
|---|---|
| Display QR on profile page | MVP |
| Download QR as image | MVP |
| Share via WhatsApp | Later |
| Apple / Google Wallet | Much later |
| Print member card | Later |
| Screenshot support | Automatic (no action needed) |

### Q4 — Permanent or time-limited?

**Permanent for MVP.** Nuemart is not a banking or payment app. A permanent QR is simpler for store staff and customers. The QR must not encode any sensitive data — only the public URL.

Rotate to time-limited QR only if/when wallet, payment, or identity verification is added.

### Q5 — Where does it live in the app?

New **My Profile** page: `/account/profile`

```
My Profile
- Name
- Email
- Customer ID (customerCode)
- My QR Code [image]
- [Download QR] button
```

Accessible via the account/header menu. Not buried in settings.

## Confirmed MVP Scope

### Customer side — `/account/profile`
- Customer name
- Customer email
- Customer ID (`customerCode`, e.g. `CUST-000124`)
- QR code (rendered via `react-qr-code`)
- Download QR button

### QR code encodes
```
https://neumart.com/qr/customer/{qrCodeId}
```

### QR scan result page — `/qr/customer/[qrCodeId]`

**Public / unauthenticated view:**
- Customer name
- Customer ID (customerCode)
- Membership / customer status

**Logged-in admin view (enhanced):**
- All of the above
- Email / phone if available
- Order count
- Last order
- Link to admin customer detail page (if built)

**Error states:**
- Invalid QR code → "QR code not found"
- QR disabled → "This QR code has been disabled"
- Customer not found → graceful fallback

## Schema Impact

New optional fields required on the `users` table in `schema.ts`:

| Field | Type | Purpose |
|---|---|---|
| `customerCode` | `v.optional(v.string())` | Human-readable customer ID, e.g. `CUST-000124` |
| `qrCodeId` | `v.optional(v.string())` | Unique public-safe token for QR URL, e.g. `nq_8F3K92LPA` |
| `qrEnabled` | `v.optional(v.boolean())` | Allows disabling QR per user if needed |
| `qrCreatedAt` | `v.optional(v.number())` | Timestamp of QR generation |

All fields are optional. Existing users without these fields must not break.

Generation logic:
- `customerCode`: auto-generated on first QR creation (e.g. sequential `CUST-NNNNNN` or short code)
- `qrCodeId`: unique random token suitable for public URLs (e.g. `nq_` prefix + nanoid)

- Apple / Google Wallet
