## Request Summary

Add a moving promotional image carousel at the top of the product listing page (`/products`, SCR-CUS-0001). Auto-scrolling with manual navigation controls (arrows + dots). MVP uses a hardcoded static banner config; architecture kept ready for future admin-managed banners.

## Original Request

> "I would need a moving carousel with images at the top of the webpage."

## Classification

- **Primary:** Existing Module Feature
- **Secondary flags:** None
- **Confidence:** High

## Module & Screen

- **Module:** Customer Commerce
- **Sub-module:** Product Listing
- **Primary screen affected:** SCR-CUS-0001 — Product Listing (`/products`)

## Grilling Answers (2026-06-22)

### 1. What goes in the carousel?
Promotional banners designed for the storefront — highlights offers, seasonal collections, featured categories, or key product groups. Must NOT auto-pull random featured product images from Convex.

### 2. Where do images come from?
**MVP:** Static banner config array hardcoded in the frontend. No new admin panel.
**Future:** Admin-managed banner module (upload, display order, active/inactive toggle, start/end dates, link target). Architecture must be designed to enable this migration without rewriting the UI.

### 3. Auto-scroll or manual?
Both. Auto-scrolls every few seconds. Users can also manually click arrows or swipe through.

### 4. Navigation controls?
- **Desktop:** Arrows + dot indicators.
- **Mobile:** Dot indicators always visible; swipe gesture works; arrows hidden on mobile if they crowd the UI.

### 5. Reference / Style
Clean grocery/e-commerce style — similar to Blinkit, Zepto, BigBasket, Instamart, Amazon Fresh. Rounded corners, good spacing, mobile-first layout. Promotional banner style, not product cards.

## Banner Config Structure (MVP)

Each banner object must include:
- `imageUrl` — banner image URL
- `title` — headline text (optional overlay)
- `subtitle` — subtext (optional overlay)
- `ctaText` — call-to-action button label
- `linkTarget` — route to navigate to on click (category, product listing, offer page)
- `isActive` — boolean to include/exclude banner without deleting config

## Scalability Contract

The static config array must mirror the schema of a future Convex `banners` table. The carousel UI component must accept banners as a prop (not import the config directly), so the data source can be swapped to Convex later without touching the component.

## Blocker / Dependency

None. No schema change required for MVP.

## Notes

Future candidate: Admin Banner Management module — Convex `banners` table + admin CRUD screen at `/admin/banners`. Not in scope for this request.
