# Personas

**Object type**: `persona`  
**Owner**: Product Manager / Designer  
**Note**: User persona objects — research-based representations of key user types. Personas keep the team aligned on who we're building for.

## Core Personas (Neumart)

- **Customer** — end-user placing grocery orders
- **Admin** — store manager managing products, orders, inventory
- **Delivery Agent** — fulfilling and tracking deliveries
- **Operations Lead** — running day-to-day operations
- **Business Owner** — monitoring performance, making strategic decisions

## When to Create/Update

- When user research reveals a new user type
- When an existing persona significantly evolves
- When a new product module introduces a new user role

## Format

`PERSONA-[NAME].md` with fields:
`id`, `name`, `role`, `goals`, `pain_points`, `tech_literacy`, `usage_frequency`, `key_scenarios`, `owner`
