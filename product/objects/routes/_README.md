# Routes

Stores route objects that map URL paths to their screens, features, file paths, and access rules. Routes are the entry points to the application.

## ID Format
`RTE-NNN`

## Linked Objects
- screens: `product/objects/screens/`
- features: `product/objects/features/`
- ui-components: `product/objects/ui-components/`

## Template
`product/os/templates/ROUTE_OBJECT_TEMPLATE.md`

## Index
`product/indexes/ROUTE_SCREEN_MAP.md`

## Policy
`product/os/policies/SCREEN_REFERENCE_RULES.md`
