# Impact Checks

Records the result of an impact analysis run on a proposed change or new pointer. Impact checks assess whether a change affects PRD, stories, design, technical design, data model, API, QA, UAT, release, or support/ops.

## ID Format
`IMP-NNN`

## Lifecycle
`pending` → `complete`

## Linked Objects
- parent change note: `product/objects/change-notes/`
- parent discovery note: `product/objects/discovery-notes/`
- parent scope change: `product/objects/scope-changes/`

`product/os/templates/IMPACT_CHECK_TEMPLATE.md`
