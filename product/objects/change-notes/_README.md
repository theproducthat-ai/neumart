# Change Notes

Records changes discovered or introduced to product objects during PRD writing, design, development, QA, UAT, or release. A change note explains what changed, why, and what the impact is.

## ID Format
`CHN-NNN`

## Lifecycle
`draft` → `reviewed` → `applied` | `rejected`

## Linked Objects
- parent object (PRD, feature, story, release, etc.)
- scope changes: `product/objects/scope-changes/`
- impact checks: `product/objects/impact-checks/`

## Template
`product/os/templates/CHANGE_NOTE_TEMPLATE.md`

## Policy
`product/os/policies/MIDSTREAM_CHANGE_RULES.md`
