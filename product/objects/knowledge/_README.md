# Knowledge

**Object type**: `knowledge`  
**Owner**: Product Manager  
**Note**: Knowledge base objects — documents that capture important product, domain, or team knowledge that doesn't fit neatly into another object type. Think of this as the product team's internal wiki.

## When to Use

- Documenting domain knowledge (how grocery delivery works, industry benchmarks)
- Capturing onboarding information for new team members
- Recording lessons learned from a project or incident
- Documenting competitive intelligence

## This is NOT for:

- Product decisions → use `decisions/`
- Process rules → use `rules/` or `os/policies/`
- Research findings → use `ux-research/`
- Historical reference → V1 numbered folders

## Format

`KNW-[TOPIC]-NNNN.md` with fields:
`id`, `title`, `category`, `content`, `source`, `last_updated`, `owner`
