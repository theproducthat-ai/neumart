# Scope Changes

Formal records of changes to the agreed scope of a feature, PRD, sprint, or release. A scope change is a change note that has been evaluated and confirmed to alter the boundaries of what was agreed.

## ID Format
`SCH-NNN`

## Lifecycle
`proposed` → `reviewed` → `approved` | `rejected`

## Approval Required
Scope changes to a feature in active development or a PRD under review require owner sign-off. See `product/os/policies/APPROVAL_GATES.md`.

## Linked Objects
- parent feature or PRD
- triggering change note: `product/objects/change-notes/`
- impact check: `product/objects/impact-checks/`
- version history: `product/versions/`
