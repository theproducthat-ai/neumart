# Backlog Items

Holds requests and ideas that are valid but not ready for active delivery. Items here are prioritised and pulled into active work during sprint/cycle planning.

## ID Format
`BLI-NNN`

## Lifecycle
`backlog` → `ready` → `in-progress` (moves to request or feature)

## Linked Objects
- source requests in `product/objects/requests/`
- template: `product/os/templates/BACKLOG_ITEM_TEMPLATE.md`
- view: `product/views/BACKLOG_VIEW.md`
